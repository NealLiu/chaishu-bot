# 拆书工作流 · 设计文档

> 状态：已确认 | 推送部分暂缓

## 概述

从网络获取书籍，AI 总结核心知识点，生成小红书风格知识卡片图片，每日推送给用户；建立自生长 Obsidian 知识库保存拆书内容，支持偏好追踪与智能推荐。

## 组件

### 1. n8n 工作流引擎
- 部署：本地 Docker 或 n8n 云
- 触发方式：Schedule Trigger (cron) + Webhook Trigger
- 核心节点：HTTP Request、Code、AI Agent

### 2. 推荐引擎（Code 节点 / AI 节点）
- 读取 `books/书籍索引.md` 中的偏好图谱 YAML
- 分析已有主题、深度、风格偏好
- 发现知识缺口 → 推荐下一本书
- 自生长逻辑：每拆一本书，更新偏好图谱

### 3. DeepSeek API（书籍拆解）
- 输入：书名 + 作者
- 输出：结构化 Markdown，包含：
  - 核心框架（一句话 + 关系图）
  - 5 个核心知识点（含洞见 + 练习）
  - 金句
  - 行动清单
  - 关联推荐（横向/深度/实用各一本）

### 4. MD2Card 渲染
- 网站：md2card.com
- 方式：API 调用或自动化脚本
- 模板：用户自选（默认"温暖柔和"）
- 输出：1242×1660px PNG，适合小红书发布

### 5. Obsidian 知识库
- 插件：Local REST API（端口 27123）
- 文件结构：
  ```
  拆书/
  ├── books/
  │   ├── 书籍索引.md          ← 偏好图谱 + 阅读路径
  │   └── YYYY-MM-DD 《书名》拆书.md
  ├── cards/
  │   └── YYYY-MM-DD 《书名》卡片.png
  └── templates/
      └── 拆书模板.md
  ```

## 数据流

```
书名 → DeepSeek (搜索+拆解) → Markdown
  → MD2Card (渲染) → PNG
  → Obsidian REST API (写入 vault)
  → 更新 书籍索引.md 偏好图谱
```

## 自生长推荐算法

```
输入: 书籍索引.md 中的 topics[], depth_level, recent_focus
输出: 推荐书名 + 推荐理由

规则:
1. 用户指定书籍 → 直接使用，分析其主题标签
2. 无指定时:
   a. 从已有 topics 向外扩展一层（交叉学科）
   b. 检查 depth_level，若<3 则推荐同领域进阶书
   c. 优先填补"知识缺口"列表中未探索的方向
3. 更新偏好图谱: 合并新书标签，调整 depth_level
```
